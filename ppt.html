1. Introducción y Motivación

Definición del problema y justificación del desarrollo del sistema.

Valor y Beneficio Global

  • Para los Usuarios: Explicación clara de la optimización del flujo de trabajo y accesibilidad a la información.
  • Para los Clientes: Retorno de inversión, escalabilidad y consistencia en la persistencia de sus datos.

2. Modelo Entidad-Relación

Supuestos

  • El transporte solo se encuentra una cantidad finita de tiempo en un local después pasa a otro.
  • Un cliente es persona o empresa, pero no ambos, la persona que represente a una empresa se representara como empresa.
  • La empresa siempre es un cliente mayorista.
  • Un empleado puede trabajar en un solo local.
  • Cada envío corresponde a un pedido ya registrado.

restricciones no Modelables

  • El stock es no negativo, no se puede vender un producto sin existencias.
  • El sistema maneja perdidas, robos y mermas como disminución del stock manual.
  • No permite cantidades negativas en pedido, envíos, clientes como cantidades estrictamente no negativas.
  • La fecha de entrega debe ser mayor o igual a la fecha del pedido.
  • No todas las boletas poseen un envio asociado, ya que el envio es solo para los Clientes tipo Empresa.

Modelo Relacional

Parte 1: Ventas y Productos

  • Compra(idCompra, rutCliente, idCajero, metodoPago)
    • FK: rutCliente → Cliente
    • FK: idCajero → Empleado
    • rutCliente, idCajero not null

  • Producto(SKU, categoria, nombreProducto, precio)

  • BoletaCompra(idCompra, SKU, cantidad, lugar, diaCompra, mesCompra, añoCompra)
    • FK: SKU → Producto
    • FK: idCompra → Compra
    • idCompra, SKU not null

Modelo Relacional

Parte 2: Distribución y Clientes

  • Local(idLocal, ciudadLocal, calleLocal, numeroLocal)

  • Ofrece(idLocal, SKU, stock)
    • FK: idLocal → Local
    • FK: SKU → Producto
    • idLocal, SKU not null

  • Cliente(rut, nombreCliente, tipoCliente)
    • Persona(rut)
      • FK: rut → Cliente
      • rut not null
    • Empresa(rut, correoCliente, razonSocial)
      • FK: rut → Cliente
      • rut not null

Modelo Relacional

Parte 3: Logística y Personal

  • Envio(idEnvio, rutCliente, idRepartidor, horaEstimada, direccionEnvio, precioEnvio)
    • FK: rutCliente → Cliente
    • FK: idRepartidor → Empleado
    • rutCliente, idRepartidor not null

  • Empleado(idEmpleado, nombreEmpleado, correoEmpleado, tipoEmpleado)
    • Cajero(idEmpleado)
      • FK: idEmpleado → Empleado
      • idEmpleado not null
    • Repartidor(idEmpleado)
      • FK: idEmpleado → Empleado
      • idEmpleado not null

Modelo Relacional

Parte 3: Logística y Personal

  • Vehiculo(patente, disponibilidad, tipoVehiculo)

  • Conduce(idRepartidor, patente)
    • FK: idRepartidor → Empleado
    • FK: patente → Vehiculo
    • idRepartidor, patente not null

DDL: Módulo Clientes

-- Tabla Cliente
CREATE TABLE Cliente(
  rut VARCHAR(10) PRIMARY KEY,
  nombreCliente VARCHAR(100),
  tipoCliente VARCHAR(30)
);
-- Tabla Persona
CREATE TABLE Persona(
  rut VARCHAR(10) NOT NULL,
  FOREIGN KEY (rut) REFERENCES Cliente(rut)
);
-- Tabla Empresa
CREATE TABLE Empresa(
  rut VARCHAR(10) NOT NULL,
  correoCliente TEXT,
  razonSocial VARCHAR(100),
  FOREIGN KEY (rut) REFERENCES Cliente(rut)
);

DDL: Ventas y Productos

-- Tabla Producto
CREATE TABLE Producto(
  SKU INTEGER PRIMARY KEY,
  categoria TEXT,
  nombreProducto TEXT,
  precio INTEGER
);
-- Tabla Compra
CREATE TABLE Compra(
  idCompra SERIAL PRIMARY KEY,
  rutCliente VARCHAR(10) NOT NULL,
  idCajero SERIAL NOT NULL,
  metodoPago VARCHAR(20),
  FOREIGN KEY (rutCliente) REFERENCES Cliente(rut),
  FOREIGN KEY (idCajero) REFERENCES Empleado(idEmpleado)
);
-- Tabla BoletaCompra
CREATE TABLE BoletaCompra(
  idCompra SERIAL NOT NULL,
  SKU INTEGER NOT NULL,
  cantidad INTEGER,
  lugar VARCHAR(30),
  diaCompra INTEGER, mesCompra INTEGER, anoCompra INTEGER,
  FOREIGN KEY (idCompra) REFERENCES Compra(idCompra),
  FOREIGN KEY (SKU) REFERENCES Producto(SKU)
);
-- Tabla Local y Ofrece
CREATE TABLE Local(
  idLocal SERIAL PRIMARY KEY, ciudadLocal VARCHAR(30), calleLocal VARCHAR(30), numeroLocal INTEGER
);
CREATE TABLE Ofrece(
  idLocal INTEGER NOT NULL, SKU INTEGER NOT NULL, stock INTEGER,
  FOREIGN KEY (idLocal) REFERENCES Local(idLocal),
  FOREIGN KEY (SKU) REFERENCES Producto(SKU)
);

DDL: Logística y Entregas

-- Tabla Vehiculo
CREATE TABLE Vehiculo(
  patente VARCHAR(6) PRIMARY KEY,
  disponibilidad BOOLEAN,
  tipoVehiculo VARCHAR(20)
);
-- Tabla Envio
CREATE TABLE Envio(
  idEnvio SERIAL PRIMARY KEY,
  rutCliente VARCHAR(10) NOT NULL,
  idRepartidor INTEGER NOT NULL,
  horaEstimada TIME, direccionEnvio TEXT, precioEnvio INTEGER,
  FOREIGN KEY (rutCliente) REFERENCES Cliente(rut),
  FOREIGN KEY (idRepartidor) REFERENCES Empleado(idEmpleado)
);
-- Tabla Conduce
CREATE TABLE Conduce(
  idRepartidor INTEGER NOT NULL,
  patente VARCHAR(6) NOT NULL,
  FOREIGN KEY (idRepartidor) REFERENCES Empleado(idEmpleado),
  FOREIGN KEY (patente) REFERENCES Vehiculo(patente)
);
